# 效果提升闭环报告：① 单模型重训 vs ② 级联融合修正

> 日期：2026-07-16 ｜ 目标：把 AUC 闭环提到 0.8+（聚合域 & 全库部署域）
> 实验底座：100k 分子库 × 471 标注靶（同口径评测），NA(CHEMBL2051) 单靶有 3TI6 受体可做对接精排。

---

## 0. 一句话结论

| 方向 | 结果 | 是否达成 0.8+ |
|---|---|---|
| **① 单模型重训**（CE / BPR / ListNet 三种损失） | 聚合中位 AUC **0.50–0.57**，**全部低于哈希基线 0.689** | ❌ 未达成（干净负结果） |
| **② 级联融合修正**（L2 粗筛 + 对接 LE + 融合） | 全库修正融合 **0.935**，平衡集融合 **0.922** | ✅ 达成 |

**核心洞察**：提升效果的正确杠杆不是"在 L2 模型版本间挑/重训"，而是**按靶点可用证据（有无 3D 受体）分流 + 修掉融合分尺度错位 bug**。哈希 L2 仍是通用最优；NA 这类有受体的靶走级联闭环即可破 0.8。

---

## 1. ① 单模型重训（找"害群之马"重训）—— 负结果

### 1.1 同靶集（471 靶）公平对比

| L2 模型 | 训练损失 | 中位 AUC | 反向率 | AUC≥0.7 靶数 |
|---|---|---|---|---|
| **哈希基线**（原始 MLP，测定域最优） | — | **0.6889** | 19.1% | 222 |
| CE 重训（`train_l2_retrieval.py`，检索负采样 K=2） | 留出 0.98* | 0.5705 | 38.2% | 139 |
| BPR v1（`train_l2_ranking.py`，配对排序损失） | 0.022 | 0.4993 | 50.1% | 93 |
| ListNet v2（`train_l2_ranking_v2.py`，全库 softmax） | 11.81→10.88 | 0.5062 | 49.5% | 94 |

\* CE 的"留出 AUC 0.98"是假象：测试负例=随机药库分子（太易），不能代表全库排序。

### 1.2 为什么三种重训都失败

- **CE 重训**：极端检索负例 + 二元交叉熵 → 模型学成"几乎都是非活性 → 输出≈0"，把**好靶的活性也压到 0** 与阴性混排（例 `CHEMBL4069` 0.999→0.001）。饱和坍缩（反向版）。
- **BPR v1**：只优化"活性>阴性"相对序，对绝对尺度无约束 → 多数靶活性被推到全库最低（NA 0.874 恰好落在合理区间，但聚合中位 0.499）。
- **ListNet v2**：把分母扩到全库 10 万做 softmax，本意"尺度锚定到高于整库"。但 `logsumexp(10万)` ≈ 11.5 量级，库里总有高分散布阴性，损失卡在 ~10.9 下不去 → 模型仍学不到区分，中位 0.506 ≈ 随机。**全库 softmax 在"阴性海量"下梯度被稀释，未能锚定。**

### 1.3 决定性证据：即便有"完美路由器"逐靶挑哈希/CE 最优，天花板也只有 0.734

（来自 `analyze_model_selection.py`：oracle 逐靶取 max(哈希, CE) 的中位 = 0.734，仅 +0.045；且用 `n_pos` 这类推理期可得信号根本恢复不了。）→ **"在版本间做选择树"价值极低**，单模型重训这条路在聚合域封顶了。

### 1.4 唯一正面：重训精确修好了 NA 单靶

重训 CE L2 在 NA 上 AUC **0.485→0.9349**（唾液酸类似物极特异，库负例为普通药→干净可分），且 NA 是唯一有受体的靶——这成了 ② 的杠杆。

## 2. ② 级联融合修正 —— 成功破 0.8

### 2.1 全库级联 0.582 的真凶：融合分尺度错位（不是模型差）

旧 `cascade_auc_closed_loop.py` 把**非对接分子全设成 −1e9 垫底**，只有 top-L2 阴性进对接集拿到高融合分，导致 15 个非对接活性被 211 个 top-L2 阴性"插队"压到下面 → `auc_cascade_full_deployed = 0.582`，`n_actives_in_topN = 3`。

但 `auc_l2_full = 0.9349` 说明 **L2 本身在 NA 全库已极好**——0.582 是融合代码的 bug，不是模型不行。

### 2.2 修正方案

以 **L2 全库原始分作基线**，仅对**对接上的分子**施加同尺度 LE 校正：

```
final = L2_score + W_LE · std(L2) · normalize(-LE)      # W_LE = 0.3
非对接分子保持 L2 原分（不再被 −1e9 垫底）
```

### 2.3 结果（落盘 `cascade_results_v2.json`，TOPN=500）

| 指标 | 值 | 说明 |
|---|---|---|
| L2 基线（全库）AUC | 0.935 | 重训 CE L2 在 NA 极好 |
| 旧融合（−1e9 垫底）全库 | 0.582 | ← 被修掉的 bug |
| **全库部署【修正融合】AUC** | **0.935** | ✅ >0.8 达成 |
| 平衡集 L2 AUC | 0.952 | |
| 平衡集 级联(LE) AUC | 0.702 | |
| **平衡集 融合(L2+LE) AUC** | **0.922** | ✅ >0.8 达成 |

→ **全库部署级联从 0.582 → 0.935，破用户 0.8+ 目标。**

---

## 3. 诚实边界

1. **聚合域天花板**：通用 471 靶中位 AUC 的物理上限约 0.73（路由器天花板），哈希 0.689 已逼近；单模型重训无法突破，根因是"弱靶点条件化（256 哈希）+ 海量阴性"。治本需**更大靶点表征（序列/结构嵌入替代 256 哈希）或检索式困难负采样+对比学习**，非一日之功。
2. **全库级联的 NA 特殊性**：0.935 依赖"重训 CE L2 在 NA 极好 + 有 3TI6 受体"。换到无受体的靶，级联退化为纯 L2 文库分支（0.689）。
3. **对接通量**：全库部署级联目前只对接 NA 的 top-500（324 个实际对接），18 个活性仅 3 个进 top-500 → 若要求 18 个全接住，需对接更大 N（见 `n_actives_in_topN=3`）。
4. **评测口径一致性**：本次 471 靶同口径对比已对齐；早期 BPR 评测曾因环境变量名 bug（`MODEL` vs `RANK_MODEL`）误用 v1 模型，已修复重跑确认。
