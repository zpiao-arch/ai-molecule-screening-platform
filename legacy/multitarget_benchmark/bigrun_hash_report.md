# 大通量闭环（哈希版 L2）报告 — 冻结版

> 日期: 2026-07-15 | 目的: 对全库 L2 从「序列嵌入」换回「靶点哈希」后重跑 100k×10k 大通量, 验证检索 AUC 是否拉回。
> 配套: `bigrun_score_hash.py`(因子化哈希 L2 推理) + 已改默认的 `scoring.py --l2 bindingdb`。

## 1. 关键结论

**「对全库修改」= 把大通量闭环 + 生产 CLI 的 L2 从 `bindingdb_seq`(序列嵌入) 换回 `bindingdb`(靶点哈希)。这一刀直接、显著地修复了检索域 AUC。**

| 指标 | 序列 L2（旧, 上一轮） | **哈希 L2（新, 本轮）** | 变化 |
|---|---|---|---|
| AUC 中位 | 0.568 | **0.689** | **+0.121 (+21%)** |
| AUC 均值 | 0.548 | **0.658** | +0.110 |
| 反向靶点 (AUC<0.5) | 184/471 = **39%** | 90/471 = **19%** | **-20pp (减半)** |
| AUC ≥ 0.7 | 125/471 = 27% | **222/471 = 47%** | +20pp |
| EF@1% 中位 | 0.0 | 0.0 | 不变（库放大必然塌陷） |
| recall@1% 中位 | 0.0 | 0.0 | 不变 |
| 规模 / 耗时 | 100k×10k / 2503s | 100k×10k / **1271s** | -49%（哈希推理更轻） |

**结论**: 之前那个 0.568 的「弱结果」, 相当一部分是我们**选错了 L2 变体**——序列嵌入在测定域强(留出 0.80)但检索域弱(0.553), 而哈希版检索域天然更优。换回哈希后, 大通量检索 AUC 中位从 0.568 拉到 **0.689**, 已接近此前 24k×164 旧面板哈希基线(0.673), 印证这是「修正错用」而非运气。

## 2. 全库修改落地位置

### 2.1 生产 CLI（`评分_work_package/评分/scoring.py`）
```python
# 修改前
parser.add_argument("--l2", default="bindingdb_seq", choices=["bindingdb_seq","bindingdb","deeppurpose"], ...)
# 修改后
parser.add_argument("--l2", default="bindingdb", choices=["bindingdb","bindingdb_seq","deeppurpose"], ...)
```
即命令行默认 `--l2` 从序列版改为哈希版。靶点文本由 `TargetResolver` 自动解析为训练一致的 `target_text`。

### 2.2 大通量科研压测（`bigrun_score_hash.py`, 新建）
- 载入 100k 分子 520 维预计算特征 + `bigrun_score.py` 同款 10k 靶点集(471 标注 + 9529 fasta 填充)
- 用 `Layer2BindingDB(prefer="mlp")` 载入哈希 MLP(776→256→128→1, 末层 logistic)
- **因子化推理**: 首层按输入拆成分子块(520)与靶点块(256), molpart 对 100k 预计算一次, tgtpart 每靶算一次后广播相加 → 免去 10 亿×776 大矩阵乘
- 手写前向与 sklearn `predict_proba` 一致性校验: max|diff| = 6.8e-6
- 每靶取 top-100 命中 + (标注靶) AUC/EF@1%/recall@1%

> 注: 因子化前向验证时踩过的坑已记入——sklearn `coefs_` 形状是 `(inp, out)`, **首层不可转置**(直接 `@ W[0]` 即可), 转置会引入 ~0.5 量级偏差。

## 4. 诚实边界（仍必须说清楚）

1. **这是「换对模型」的修复, 不是「突破天花板」。** 哈希 L2 检索 AUC 中位 0.689 已是当前训练数据下的合理上限(检索域天花板约 0.67–0.69, 见旧面板)。要再往上, 必须做检索式负采样重训(治本)或级联对接精排(治标)。

2. **EF@1% / recall@1% 依然塌到 0**, 不是回归。10 万库的 top1% = 1000 位, 而每靶中位仅 ~25 个活性, 弱排序器把活性塞不进顶端。AUC 是规模不变量(放大 255 倍仍 0.69), EF/recall 是库大小的函数, 跨规模不可比。**大通量评测以 AUC 为主指标**。

3. **NA(3TI6) 特例仍成立**: 此前归因已证明 NA 的 L2 分数**整体坍缩**(47% 分子 >0.9), 对 NA 检索近乎随机排序, 不能当预筛。哈希版换回后 NA 检索 AUC 同上(中位 0.485 是全库视角下的特例靶点, 不进 471 高 AUC 主体)。**NA 这类靶点必须靠对接精排②**——已验证级联 0.413→0.765。

4. **10k 靶点里只有 471 个有标签可算 AUC**, 其余 9.5k 是 fasta 占位, 仅贡献吞吐与漏斗规模。
