# 四级 CLI 1000x10000 闭环验证设计

## 1. 目标

在本机对 1,000 个真实药物靶点分别运行 10,000 个候选分子，完成 10,000,000 个靶点-分子对的四级评分，并分别回答两个问题：

1. 工程闭环：正式 CLI 的 L1、L2、L3、L4、综合分、质量门槛和可用受体级联能否在固定环境中稳定运行并生成可复算证据。
2. 科研效果：剔除模型拟合样本后，L2 和四级综合分在严格留出标签上的 AUC，以及在 10,000 候选背景中的早期召回表现如何。

工程吞吐和科研效果必须分开报告。未标注靶点可以贡献吞吐，但不得进入聚合 AUC。

## 2. 非目标

- 不声称 1,000 个靶点都完成结构对接。当前注册表仅有 NA/CHEMBL2051 的 3TI6 受体。
- 不把随机背景分子称作确认阴性。
- 不用评测标签在运行后选择更高 AUC 的模型或分支。
- 不使用 L4=0、ADMET proxy、占位 docking 或其他静默降级完成“全套”验收。
- 不以本次计算替代湿实验或外部时间切分验证。

## 3. 评测对象与数据契约

### 3.1 靶点集合

靶点按以下优先级选择，最终必须恰好 1,000 个不同的真实 ChEMBL 靶点 ID：

1. 在重建的模型测试划分中同时具有至少 5 个正例和 5 个确认阴性的靶点。
2. 只有留出正例、但确认阴性不足的真实靶点。
3. 仍不足 1,000 时，从具有可解析 target_text 或 UniProt 序列的 ChEMBL 靶点补齐。

每个靶点记录 `target_id`、`target_text`、解析来源、序列可用性、受体可用性和评测资格。第 2、3 类只计算吞吐或有限指标，不进入严格 AUC 聚合。

### 3.2 训练划分重建与污染控制

生产 L2 模型由 BindingDB 和 ChEMBL 对齐样本合并、按 `(canonical_smiles, target_text)` 去重后，以 `train_test_split(test_size=0.2, random_state=0, stratify=y)` 训练。验证程序必须重建同一划分并输出：

- `fit_pairs`: 实际拟合侧的靶点-分子键集合。
- `heldout_positive_pairs`: 测试侧正例。
- `heldout_negative_pairs`: 测试侧确认阴性。

任何用于效果评测的标签对都不得出现在 `fit_pairs`。运行前和运行后都执行污染审计，验收条件为直接拟合对重叠数等于 0。

该划分仍允许同靶点和同分子跨对出现，因此只代表 pair-heldout 能力；报告必须明确它不是冷靶点、冷分子、骨架或时间外推。

### 3.3 每靶 10,000 候选池

每个候选池按固定顺序构建后再用种子打乱：

1. 该靶点全部合格留出正例。
2. 该靶点全部合格留出确认阴性。
3. 从冻结的 100,000 分子库中无放回抽取未标注背景，补足到 10,000。

排除规则：

- 排除该靶点拟合侧的所有标签对。
- 排除重复 canonical SMILES。
- 排除 RDKit 无法解析的分子。
- 若标签分子不在冻结库中，记录为 `label_not_in_library`，不得悄悄替换。

每个分子记录 `label_role=heldout_positive|heldout_negative|unlabeled_background`。随机背景不参与严格 AUC 的负类定义。

## 4. 批量 CLI 架构

采用“正式实现 + 目标无关层缓存 + 小样本等价性证明”的批量 CLI，而不是启动 1,000 个逐分子子进程。

### 4.1 固定运行环境

创建独立的四级验证环境并冻结：

- Python 版本。
- RDKit、NumPy、scikit-learn、PyTorch、UniMol、pyarrow/joblib 版本。
- smina、obabel 二进制路径和 SHA-256。
- L2、ADMET、UniMol、参考嵌入及受体资产 SHA-256。

环境预检必须真实加载全部模型并完成一个分子的四层评分。任一必需依赖或资产缺失时硬失败。

### 4.2 正式 CLI 修复

对当前 CLI 做限定修复：

- 让 `pipeline_router.py` 在受支持的 Python 版本上可导入，并为路由结果增加资产存在性验证。
- 为 L4 增加 strict 模式；UniMol 加载失败时返回非零退出码，不再默认为 0。
- 固定模型反序列化兼容版本，消除 scikit-learn 跨版本警告。
- 将级联改为真正的两阶段语义：四层/L2 粗筛后只对 top-N 做 smina，而不是对全部输入先对接。
- 增加 `--cascade-top-n`，本次冻结为 300。
- 所有层输出显式 `status`、`backend` 和 `model_asset_id`。

### 4.3 目标无关层缓存

对冻结 100,000 分子库只计算一次：

- L1：RDKit 分子质量、规则和 `layer1_score`。
- L3：真实 ADMET 模型和规则的完整字段。
- L4：真实 UniMol 表征与 FDA 参考集相似度。

缓存键为 canonical SMILES 与模型/代码资产哈希。缓存必须包含失败状态，不能把失败替换成 0。

### 4.4 L2 与四层聚合

对每个靶点的 10,000 候选池运行与生产 `Layer2BindingDB` 数学等价的批量前向。随后复用正式 CLI 公式：

```text
final_score = 0.20 * L1 + 0.50 * L2 + 0.20 * L3 + 0.10 * L4
```

质量门槛必须调用与正式 CLI 相同的规则。批量路径不得复制一份会漂移的权重或门槛常量；公式和门槛应提取为共享 API。

### 4.5 级联对接

路由决策只依赖运行前已知资产：受体注册表中存在且文件/盒参数有效时进入 cascade，否则进入 library。不得读取评测标签或 AUC 决定分支。

NA/CHEMBL2051 的流程为：

1. 按冻结的四层基线分或明确记录的 L2 粗筛分选 top-300；本次默认使用 L2，以与既有级联定义一致。
2. 对 top-300 运行真实 obabel 3D 制备和 smina。
3. 记录 `ok`、`skipped_hac`、`prep_failed`、`timeout`、`dock_no_score` 等逐分子状态。
4. 只对成功分子应用冻结的 LE 修正，未成功分子保留基线分。
5. 重新计算融合后的 AUC、EF 和 recall，不沿用融合前指标。

## 5. 等价性验证

全量运行前，使用至少 20 个覆盖正常、无效、毒性、高分子量和 NA 抑制剂的分子做小样本对照：

- 正式 `scoring.py` 单次 CLI 输出。
- 批量 CLI 输出。

逐字段比较 L1、L2、L3、L4、final_score 和 gate，数值容差 `1e-4`。路由分支、模型资产和状态必须完全一致。级联另用小规模 top-N=3 验证对接选择与融合重算。

只有等价性测试通过后才能启动 1,000x10,000。

## 6. 指标

每个有资格靶点分别输出：

- `auc_l2_labeled`: 仅严格留出正例 vs 严格留出确认阴性。
- `auc_final4_labeled`: 同一标签集合上的四级综合分 AUC。
- `recall_at_1pct_l2`、`recall_at_1pct_final4`: 10,000 全池 top-1% 命中留出正例比例。
- `observed_enrichment_at_1pct`: 相对池内已知正例基线的观察富集；明确属于 positive-unlabeled 口径。
- PASS/FAIL 比例和各层失败率。
- 对接前后独立重算的相同指标。

聚合报告给出中位、均值、四分位数、bootstrap 95% CI、可评测靶点数和不合格原因分布。不得把只有 1 个受体的对接结果推广到 1,000 靶点。

## 7. 产物与可复算性

产物目录使用带时间戳的冻结目录，至少包含：

- `DESIGN.md`/设计文档引用和运行配置。
- `environment_manifest.json`：运行时与资产哈希。
- `target_manifest.parquet`：1,000 靶点与评测资格。
- `pool_manifest/`：每靶 10,000 候选及 label_role。
- `scores/`：10,000,000 行候选级 L1/L2/L3/L4/final/gate/status，按 target 分区压缩。
- `docking/`：逐分子命令参数、状态、affinity、LE，不保存秘密或认证信息。
- `per_target_metrics.parquet`。
- `summary.json` 和中文审计报告。
- `run.log`、阶段 checkpoint 和失败清单。
- `MANIFEST.sha256`。

汇总指标必须能仅凭 `scores/`、`target_manifest` 和配置重新计算，不依赖 `/tmp` 中未归档状态。

## 8. 容错与恢复

运行分为可恢复阶段：预检、划分与建池、L1/L3/L4 缓存、L2 分片、四层聚合、对接、指标、归档。每个阶段完成后写原子 checkpoint 和输入哈希。输入或代码哈希变化时拒绝错误续跑。

单分子可解释失败写入候选状态并继续；模型整体加载失败、资产哈希变化、污染不为 0、候选池不满 10,000 或等价性失败时停止全量运行。

## 9. 验收标准

1. 靶点数恰好 1,000，每靶候选数恰好 10,000，总行数恰好 10,000,000。
2. 每行都有 L1/L2/L3/L4/final/gate 及各层状态；L4 不允许因依赖失败批量为 0。
3. 严格效果标签与拟合对直接重叠数为 0。
4. 小样本正式 CLI 与批量 CLI 在规定容差内等价。
5. 路由只依赖受体资产；不存在按评测 AUC 事后选分支。
6. NA top-300 对接逐分子状态完整，融合指标重新计算。
7. 所有摘要可从候选级产物重算并通过校验脚本。
8. 报告明确区分工程吞吐、pair-heldout 效果、PU 早期检索和单靶对接结论。
